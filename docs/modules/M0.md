# M0 — Foundation & Infra (delivery note)

**Status:** ✅ Complete & verified · **Date:** 2026-06-24
**Scope:** the buildable foundation every later module sits on (backlog A7 → M0).

## What shipped
- **pnpm monorepo** (Node 20+, TS 5.6, strict) with workspaces: `apps/web` + `packages/{config,db,core,ai,integrations}`.
- **`@gns/config`** — Zod-validated env (`loadServerEnv`/`loadPublicEnv`), domain constants (case statuses, roles, review areas, agents), and the per-entity white-label config schema (`EntityConfig`).
- **`@gns/db`** — Drizzle ORM + postgres-js client; foundation schema (enums + `entities`, `users`, `roles`, `user_roles`, `audit_logs`, `events`); generated SQL migration in `packages/db/migrations`.
- **`@gns/core`** — typed error hierarchy (`AppError` → API envelope), the **onboarding state machine** (`canTransition`/`assertTransition` + guards), and the **hexagonal ports** (CompaniesHouse, Ledger, Mailer, ESign, Kyc, DocExtraction).
- **`@gns/ai`** — agent framework contract (`AgentDefinition`, confidence + HITL policy types) for the 8 agents.
- **`@gns/integrations`** — adapter registry slots for the ports (adapters land M4–M7).
- **`apps/web`** — Next.js 14 App Router with `(client)/(staff)/(admin)` portal shells, Tailwind + **per-entity theming** (CSS-variable themes per `data-entity`), a base `Button` primitive, `/api/health`, edge `middleware` (correlation id), and a server logger (pino) + error seam (Sentry-ready for M14).
- **Infra:** GitHub Actions CI (install → typecheck → lint → test → build), `supabase/config.toml`, `.env.example`, git repo initialised (`main`).

## How to run
```bash
pnpm install
cp .env.example .env.local
pnpm dev            # http://localhost:3000  → /, /client, /staff, /admin, /api/health
pnpm db:generate    # regenerate migration after schema changes
```
Routes render; `/api/health` returns `{ status: "ok", configured: {...} }` (booleans only, no secrets).

## Deliberately deferred (by design, to later modules)
- **RLS policies & DB grants** (append-only audit enforcement) → **M1** (with auth + entities CRUD).
- **Real auth** (Entra SSO / Supabase) — portals are currently unguarded shells → **M1**.
- **Domain tables** (clients, cases, documents, compliance, reviews, AI, comms) → **M2–M9**.
- **Sentry SDK activation** (seam in place) → **M14**.
- **Provider adapters** (CH, Xero/QBO, KYC, e-sign, OCR, mailer) → **M4–M7**.

## Notes / decisions
- Workspace packages ship as TS source and are compiled by Next via `transpilePackages`; server-only deps (`pino`, `postgres`) are externalised.
- `user_roles` grant uniqueness enforced via a unique index `uq_user_role_entity`.
- Build is required to pass **without real secrets** so CI and preview deploys work pre-credentials.
