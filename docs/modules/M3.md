## What shipped

### Database (`packages/db`)
- **New enums** (`0004` migration): `document_status` (pending/received/verified/rejected), `document_category` (10 values), `classification_source` (auto/manual/agent).
- **New tables** (`0004` migration):
  - `documents` — logical document record (entity+case+client scoped, links to checklist item, stores storage path + hash + mime + status).
  - `document_versions` — append-only upload history; each re-upload creates a new version row.
  - `document_classifications` — AI + manual classification results (confidence numeric, needsReview flag, confirmed-by staff override).
- **RLS policies** (`0005` migration): entity-scoped for staff, own-case for clients, insert-only propagation for versions and classifications.
- **Repository** (`packages/db/repositories/documents.ts`): insert/get/update for all 3 tables, `linkDocumentToChecklist` (updates both document and checklist item atomically), `confirmClassification`.

### Integrations (`packages/integrations`)
- **Azure Doc AI stub adapter** (`src/adapters/doc-extraction.stub.ts`): implements `DocExtraction` port; returns plausible stub fields in dev mode without a live Azure endpoint. Registered in `adapters` registry. Real Azure adapter ships in M5.

### AI agents (`packages/ai`)
- **Document Classifier** (`src/agents/classifier.ts`): Haiku-tier, threshold 0.85, HITL → Reviewer when below threshold. Two deterministic validators (confidence-range, needsReview-threshold alignment).
- **Missing-Info Detector** (`src/agents/missing-info.ts`): Haiku-tier, threshold 0.0 (always auto-accept — advisory output), HITL kind `never`.
- **Agent runner** (`src/runner.ts`): uses `@anthropic-ai/sdk` tool-use pattern for structured JSON output; applies validators (confidence docked per failure); dev stub when `ANTHROPIC_API_KEY` unset; minimal Zod→JSON Schema converter (full `zod-to-json-schema` in M8).

### Core services (`packages/core`)
- **Document service** (`src/services/documents.ts`):
  - `getUploadUrl` — returns Supabase Storage signed URL (placeholder in dev mode).
  - `recordUpload` — atomic: document + version + checklist link + audit + outbox event.
  - `listDocuments` — with latest classification attached.
  - `storeClassification` — persists agent output, updates document category + status.
  - `verifyDocument` — staff marks verified, records verifiedBy + verifiedAt + audit.
  - `overrideClassification` — staff confirms/corrects AI category + audit.
  - `getMissingDocuments` — deterministic gap analysis (blocking = required + missing, non-blocking = optional pending).
- **RBAC** — added `verify_documents` capability (Partner/Manager/Reviewer/ComplianceOfficer). `upload_documents` and `view_documents` already present from M2.

### API routes (`apps/web`)
- `GET  /api/v1/cases/:id/documents` — list documents with classifications
- `POST /api/v1/cases/:id/documents` — record upload + trigger checklist link
- `POST /api/v1/cases/:id/documents/upload-url` — get signed upload URL
- `PATCH /api/v1/cases/:id/documents/:docId` — verify or override classification
- `GET  /api/v1/cases/:id/documents/gaps` — missing document gap analysis

### Staff UI (`apps/web`)
- **Document Centre** (`/staff/cases/:id/documents`):
  - Missing-documents panel (amber warning with blocking + optional lists).
  - Document list with status badges, MIME/size info, AI classification + confidence.
  - Inline classification override form (select correct category → confirm).
  - Per-document "Verify" button.
  - "Record upload" form (links to checklist item, sets category).
  - Full checklist progress summary with strikethrough for satisfied items.
- **Case workspace** — added "Documents" link button to `/staff/cases/:id`.

## Architecture notes

- **Upload flow (production):** browser → `POST /upload-url` (get signed URL) → PUT directly to Supabase Storage → `POST /documents` (record metadata). Supabase bucket name: `documents`. Signed URLs expire in 300 s.
- **Classification flow:** after `recordUpload`, the caller (or background worker in M12) calls `runAgent(documentClassifierAgent, { documentId, label, mimeType, extractedText? })`, then `storeClassification` with the result. High-confidence → auto-verified; low-confidence → `needsReview = true` → appears in Document Centre for staff override.
- **Gap analysis:** deterministic check in `getMissingDocuments`; AI interpretation layer via `missingInfoDetectorAgent` is wired in M8 (runs on demand before `docs_complete` transition).
- **No circular deps:** `@gns/ai` does not import `@gns/core`; `@gns/core` does not import `@gns/ai`. Agent tests live in `@gns/ai`; gap-logic tests in `@gns/core`.
