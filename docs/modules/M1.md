# M1 — Identity, RBAC & Multi-entity (delivery note)

**Status:** ✅ Code complete & verified (static/build/test) · ⏳ live-DB run pending a database
**Date:** 2026-06-24 · **Approach:** local **dev-auth shim** behind the production hybrid-auth interface; **placeholder** GNS/LLP/GXY config (your choices).

## What shipped
- **Auth session model** (`@gns/core/auth`): `AuthSession` (roles, entityIds, clientId, isAdmin) + `IdentityProvider` contract. Same shape the real Entra (staff) / Supabase (client) adapters will produce.
- **Dev-auth shim** (`apps/web/lib/auth`): HMAC-signed session cookie, `getSession`/`requireSession`/`requireCapability`, a `/dev-login` role switcher (disabled in production), and `logout`.
- **RBAC** (`@gns/core/rbac`): the PRD §7.2 capability×role matrix, `can()` + `authorize()`. **5 unit tests**.
- **RLS foundation** (`packages/db/migrations/0001_rls_foundation.sql`): session helper functions (`current_user_id/current_client_id/current_entity_ids/is_admin/has_role`), RLS enabled with read/write policies on `entities/users/user_roles/events`, and **append-only** `audit_logs` (SELECT+INSERT only, UPDATE/DELETE revoked).
- **Session→RLS bridge** (`@gns/db withSession`): runs each use-case in a transaction that sets the `app.*` GUCs the policies read.
- **Entities service** (`@gns/core/services/entities`): `list/get/create/update`, each authorised + RLS-scoped + **audited atomically**.
- **Seed** (`@gns/db/seed`): 8 roles + GNS/LLP/GXY realistic placeholder config (branding, address, bank, signatory, AML supervisor).
- **API** (`/api/v1/entities` GET/POST) with the A4 §4 error envelope (`apiError`, `unauthorized`).
- **Admin UI**: entities list + create form (Admin/Partner gated), graceful "DB not reachable" state; `SessionBar` (who am I + sign out) on staff/admin shells.

## How to run
```bash
pnpm dev                      # http://localhost:3000
# → /dev-login : pick a role (Admin/Partner/.../Client) to get a session
# → /admin     : entities list + create (Admin/Partner)
# → /staff     : staff shell with session bar
```
When a database is available (local Supabase or hosted EU project), complete the live run:
```bash
# set DATABASE_URL (e.g. supabase local: postgresql://postgres:postgres@127.0.0.1:54322/postgres)
pnpm db:migrate                 # applies 0000 + 0001 (tables, RLS, audit)
pnpm --filter @gns/db seed      # seeds roles + GNS/LLP/GXY
```
Then `/admin` lists the three seeded entities and create/list works end-to-end.

## Notes / decisions
- **RLS enforcement:** policies are defined and wired to session GUCs via `withSession`. They enforce in production (Supabase non-superuser `authenticated` role); a **local superuser** connection bypasses RLS by design, so in dev the **application service layer** (`authorize()`) is the active gate — defence in depth (A2 §10).
- **Append-only audit:** no UPDATE/DELETE policy on `audit_logs`, plus grant revokes for Supabase `authenticated`/`anon`.
- Placeholder entity bank/AML/signatory details are clearly fake (`00-00-00`, `C0000001`) — replace via the admin UI or by editing the seed (PRD §18).
