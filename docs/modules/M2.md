# M2 — Client master, services/pricing, onboarding state machine (delivery note)

**Status:** ✅ Code complete & verified (static/build/test) · ⏳ live-DB run pending a database
**Date:** 2026-06-24

## What shipped
- **Domain schema** (migrations `0002` tables, `0003` RLS): `clients`, `client_contacts`, `services`, `client_services`, `pricing_agreements`, `onboarding_cases`, `case_transitions`, `checklist_items`, `task_templates`, `tasks` + matching enums. Entity-scoped RLS for staff, own-record for clients.
- **Repositories** (`@gns/db/repositories/onboarding`): typed data access for all of the above + outbox events.
- **Services** (`@gns/core`):
  - `createLead` — client + contact + case (generated `GNS-2026-0001` reference) + base CDD checklist + first task + opening transition + audit + `lead.created` event, **atomically**.
  - `selectServices` — records client services, extends the checklist from each service's required documents, transitions `lead → service_selection`.
  - `acceptPricing` — records the pricing agreement, transitions `service_selection → pricing_agreed`.
  - `transitionCase` / `applyTransition` — guarded state-machine transitions (assert legality → update → append transition → audit → emit event).
  - `listCasesForSession`, `getCaseDetail` (case + client + checklist + services).
- **Outbox** (`@gns/core/events`): `emitEvent` (in-transaction) + `dispatchPendingEvents` (default logger; n8n delivery in M12).
- **API** (A4): `POST /leads`, `GET /cases`, `GET /cases/{id}`, `POST /cases/{id}/transition` (409 on illegal), `POST /cases/{id}/services`, `POST /cases/{id}/pricing/accept`, `GET /services`.
- **Staff UI**: case **pipeline** list, **new-client** intake form, and a **case workspace** (status, advance button, service selection, pricing capture, checklist).
- **Seed**: 8-service catalogue (bookkeeping, VAT, PAYE, CIS, accounts, SA, CT, confirmation statement) with required-documents + clearance flags.

## Notes / decisions
- The app state machine is authoritative; transitions emit outbox events for n8n (M12) without n8n being a DB writer.
- Pricing is **staff-entered** in v1 (auto-quoting deferred); model field supports fixed/tiered/custom.
- Case reference `${ENTITY}-${YEAR}-${0001}` via per-entity count (unique-constrained; revisit for high-concurrency later).
- Dev session user ids are valid UUIDs so writes to uuid columns succeed on a real DB.
